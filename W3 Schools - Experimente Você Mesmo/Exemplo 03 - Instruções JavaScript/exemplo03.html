Boas práticas JavaScript


Ponto e vírgula

Terminar instruções com ponto e vírgula não é obrigatório, mas altamente recomendável.

Espaços em branco em JavaScript

Uma boa prática é colocar espaços em branco em volta dos operadores ( = + - * / ).

Comprimento de linha e quebras de linha em JavaScript

Para melhor legibilidade, gerealmente se evita linhas de código com mais de 80 caracteres. Se a instrução JavaScript não couber em uma linha, o melhor lugar para quebrá-la é depois de um operador.

Blocos de código JavaScript

As instruções podem ser agrupadas em blocos, dentro de chaves {}. Utiliza-se recuo dentro dos blocos.
